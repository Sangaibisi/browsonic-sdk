# Changelog — browsonic-sdk (monorepo root)

This file documents **cross-package, monorepo-level** changes —
schema-wide alignment programs, build-pipeline reshuffles, and
audit closures that touch multiple workspaces in lockstep. Per-package
release notes (one per published `@browsonic/*` package) are
auto-generated by `semantic-release` from Conventional Commits and
live in `packages/<name>/CHANGELOG.md`. **Do not edit those by hand.**

Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

### S0–S5+ summary

Closes 14/15 audit gaps from the 12-parallel-deep-dive review;
C1 closed in S5+ (frozen API contracts + 7 dashboard pages rewired to
`useQuery + placeholderData: MOCK_*`).

### Added — `@browsonic/sdk` core (`packages/sdk/`)

- `EventBatch.visitorId` carried on every batch (was: pageview-ping
  only); persisted via Postgres + ClickHouse.
- Opt-in `webVitalsPlugin()` exporting LCP/FCP/CLS/TTFB via native
  `PerformanceObserver` (~1 KB; FID/INP deferred to a follow-up release).
- `console.debug` capture (`collectors/console.ts`) — telemetry
  preserves the original verb via the new `methodToTelemetryLevel`
  map; `console_debug` is a first-class `EventType`.
- `utils/redaction.ts` — header allowlist + JWT / email / credit
  card / oauth secret regex patterns. Wired into both `network.ts`
  and `xhr.ts` collectors via `buildNetworkDetail` /
  `buildXhrNetworkDetail` (sizes, traceparent, abort detection).
- `sentinel/adapter-registry.ts` — module-singleton registry that
  every adapter package calls into at module load. Active
  adapter snapshot rides on every `EventBatch.adapter`.
- `sentinel/plugins.ts` — `collectPluginHealth()` + activate /
  deactivate hooks; rolls plugin telemetry into every
  `EventBatch.plugins`.
- `diagnostics/retry-tracker.ts` — p50/p95/max retry attempt
  percentiles; `permanent_fail` `DroppedReason` distinct from
  `transport_fail`. Exposed via the diagnostics aggregate.
- `widget/interaction-reporter.ts` + renderer `onInteraction` hook
  — consent-gated `POST /v1/widget-rules/{ruleId}/interactions`
  beacon for impression / click / dismiss events.
- `EventBatch.queueMetrics` + `lastFlushTimeMs` +
  `dropsSincePreviousBatch` — per-batch deltas (not cumulative)
  for the dashboard's `<QueueHealthPanel>`.

### Added — adapters (`packages/{react,vue,svelte,angular,nextjs,astro,remix}/`)

- Each adapter ships a `scripts/stamp-version.mjs` that reads
  `package.json` and writes `src/__pkg.ts` with `PACKAGE_NAME`
  and `PACKAGE_VERSION`. The `prebuild` hook runs the stamper
  before tsc; the runtime now reports the real semver after every
  `semantic-release` publish.
- `src/index.ts` calls `registerAdapter({ name: PACKAGE_NAME, version: PACKAGE_VERSION })`
  at module load — no hardcoded names or versions in the bundle.

### Added — documentation

  `WebVitalMetric` / `PluginHealthSummary` / `QueueMetricsSnapshot` /
